Adding a new host

Run the following to seed SSH host keys for a new backup source. Defaults to port 22.

sudo -u fsbackup /opt/fsbackup/utils/fs-trust-host.sh <hostname> [port]

Then add the host's targets to /etc/fsbackup/targets.yml and reload the web UI.

Renaming a target

To rename a target and all of its snapshots across tiers and mirrors, run the following as root. Always use --dry-run first.

sudo /opt/fsbackup/utils/fs-target-rename.sh \
  --class <class1|class2|class3> \
  --from <old-target-id> \
  --to   <new-target-id> \
  --move | --delete \
  [--dry-run]

--move — rename the directory in place (keeps snapshot history)

--delete — wipe all snapshot directories for that target across every tier

--dry-run — preview changes without modifying anything

Troubleshooting: Node Exporter permissions

If Prometheus metrics stop updating, the textfile collector directory may have wrong ownership or ACLs. Run as root:

sudo /opt/fsbackup/utils/fs-nodeexp-fix.sh [--web-user <username>]

The --web-user flag also grants read ACLs (including default ACL for new files) to the web UI user. Example: sudo /opt/fsbackup/utils/fs-nodeexp-fix.sh --web-user fsbackup
