Utilities

Manual administrative tools — run from the command line as fsbackup or root

Trust Host

Seed SSH host keys for a new backup source host into the fsbackup known_hosts file

utils/fs-trust-host.sh 
sudo -u fsbackup /opt/fsbackup/utils/fs-trust-host.sh <hostname> [port]

Run once when adding a new host. Defaults to port 22. Emits a Prometheus metric on completion.

Rename Target

Rename a target ID across all snapshot tiers on primary and mirror storage

utils/fs-target-rename.sh 
sudo /opt/fsbackup/utils/fs-target-rename.sh \
  --class <class1|class2|class3> \
  --from <old-target-id> \
  --to   <new-target-id> \
  --move | --delete \
  [--dry-run]

--move — rename the directory in place (keeps snapshot history)

--delete — remove the old target's snapshot directories entirely

--dry-run — preview changes without modifying anything

Annual Mirror Check

Verify that annual snapshots on the primary drive are present and matching on the mirror

utils/fs-annual-mirror-check.sh 
sudo -u fsbackup /opt/fsbackup/utils/fs-annual-mirror-check.sh

Run after the January 5 annual promotion. Compares byte counts between /backup/snapshots/annual and /backup2/snapshots/annual and emits a Prometheus metric.

Fix Node Exporter Permissions

Repair ownership, group ACLs, and default ACLs on the node_exporter textfile collector directory

utils/fs-nodeexp-fix.sh 
sudo /opt/fsbackup/utils/fs-nodeexp-fix.sh [--web-user <username>]

--web-user <username> — also grant read ACLs (including default ACL for new files) to the web UI user

Example: sudo /opt/fsbackup/utils/fs-nodeexp-fix.sh --web-user fsbackup

Restore

Restore files from a snapshot to a local path or push to a remote host over SSH

utils/fs-restore.sh 
# List available snapshots
fs-restore.sh list --type <daily|weekly|monthly> [--class <class>] [--date <key>]

# Restore locally
fs-restore.sh restore --type daily --class class2 --id nginx.config \
  --latest --to /tmp/restore-nginx

# Restore to remote host
fs-restore.sh restore --type daily --class class2 --id bind.named.conf \
  --date 2026-01-29 --to-host ns1 --to-path /tmp/restore-bind

The Restore page provides a UI for local restores with dry-run preview.
